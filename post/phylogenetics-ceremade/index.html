Phylogenetic analyses on the CEREMADE server

Luke Kelly
Last updated on Apr 26, 2021

Install and run BEAST 2

From your home directory on the cluster, download a tarball of BEAST 2 (v2.6.3 was the latest at the time of writing) and extract the files.

wget https://github.com/CompEvol/beast2/releases/download/v2.6.3/BEAST.v2.6.3.Linux.tgz
tar -xf BEAST.v2.6.3.Linux.tgz

This will create a folder beast/ in the current directory.

Next, install all of the add-on packages.

for BEAST_PACKAGE in $(java -cp beast/lib/beast.jar beast.util.PackageManager -list | tail -n +5 | cut -d" " -f1); do
    java -cp beast/lib/beast.jar beast.util.PackageManager -add $BEAST_PACKAGE
done

We can then run BEAST 2 on an XML file, say example.xml by submitting the command

java -Djava.library.path=beast/lib/ -jar beast/lib/launcher.jar example.xml

to the queue. Instructions for submitting jobs to the SLURM queue are here.

Compute posterior distributions on topologies

I use treeloganalyser in BEAST for this. First download and untar the latest release.

wget https://github.com/beast-dev/beast-mcmc/releases/download/v1.10.4/BEASTv1.10.4.tgz
tar -xf BEASTv1.10.4.tgz

Then create a file called treeloganalyser in BEASTv1.10.4/bin/ containing the script on this page.

Assuming we are in the same directory as BEASTv1.10.4 and the file output.trees from a BEAST analyses, we can run treeloganalyser with

bash BEASTv1.10.4/bin/treeloganalyser output.trees

to produce a posterior summary on trees and clades.

If we want to obtain the actual posterior support figures then we first export the treeloganalyser output then use a regular expression to get the figures we want.

bash BEASTv1.10.4/bin/treeloganalyser -export tla-report.log output.trees
sed -n 's/^.*\[\&W \(.*\) \].*$/\1/p' tla-report.log > tla-support.txt

Or at least we can using the version of sed on the CEREMADE cluster.

If the output file came from TraitLab, which uses a different naming system for outputs, then we need to edit the tree names before running treeloganalyser.

sed 's/\(tree\)\([0-9]\)/\1_\2/' tl-output.nex > tl-output.trees

SPR distances between output trees

First download Chris Whidden’s rspr and run make.

Then edit rspr/R/rspR.r to point to the location of the rspr binary; in my case I replaced instances of command = "rspr" with command = "/home/users/kelly/rspr/rspr".

I then compute SPR distances between trees in R with

source("/home/users/kelly/rspr/R/rspR.r")
t <- ape::read.nexus("output.trees")
d <- rspr.matrix(t, "full", Inf)
